this is unlike the file processing , where data is not centralized and kept

TYPES OF DATABASE SYSTEM


  • View level
  • Logical
  • Physical

  • VIEW LEVEL :
    this describes data at the highest level of data abstraction that is the view level. this type of database can be ERD or entity relationship diagram and OODBMS or object oriented database
    ERD
    consists of entities and their associated characteristics or attributes and relationships . some of extended ER properties are specialization , generalization and aggregation .
    this type of database system shows data abstraction , polymorphism and attribute inheritance . there is one level of data abstraction present.
    while specialization is represented through ISA , generalization is simple inversion to specialization
    it is not possible to express nested relations in RDBMS due to aggregation that does not support nested queries
  • oodbms : OBJECT ORIENTED DATABASE SYSTEM ---- variant of the view level. consists of objects and classes where class is a containership and class instance is an object . shows inheritance , data abstraction and attribute inheritance . there are two levels of data abstraction present
  • LOGICAL : describes data at logical and view levels . can be :
    relational database : consists of data organized in tables and identified through key values . the intersection of a row and column is a tuple and the tabular representation shows integrity constraints as entity , referential and domain integrity .
  • Network data model : here data is organized as linked lists and any arbitrary graph representation is possible .
  • Hierarchical data model : consists of data organized as fixed format records in linked lists and there are pointers to identify lists . any arbitrary graph representation is not possible
    only tree representation is possible . consists of virtual records . VIRTUAL RECORD is a data piece that contains no data item but includes a pointer to the first physical record . when data is to be replicated in several database trees , we keep a copy of that record in one of the trees and replace every other copy with a virtual record
  • PHYSICAL : describes data at the physical level of data abstraction. can be frame memory model etc

SQL or structured query language is database language and is relationally complete language
it can be :
  • data definition language : (DDL) examples : SQL commands like CREATE , DROP , INSERT etc
  • data manipulation language : (DML)
    includes SQL commands like SELECT , DELETE , UPDATE , MODIFY etc
  • DCL or data control language
    includes SQL commands like GRANT , REVOKE etc to generate authorization rights to users of database

database definitions
EXISTENCE DEPENDENCIES : if in a relation schema , the existence of entity x depends on that of entity y , it is existence dependencies and operationally if y is deleted , so is x. here x is dominant entity and y is subordinate entity

TYPES OF FUNCTIONAL DEPENDENCIES
FULL FUNCTIONAL DEPENDENCY : if in relation schema , all non key attributes are wholly dependent on the key attribute , it is full functional dependency
PARTIAL DEPENDENCY : if in schema , some non key attributes are dependent on the key attribute and the remaining non key attributes are dependent on key attribute and some other non key attribute , this functional dependency generated
TRANSITIVE DEPENDENCY : if in schema, the attributes in key column of one relation are dependent on key values in column of another relation , this holds good .
MULTI VALUED DEPENDENCY
seen in 4th normal form of relation schema
JOIN DEPENDENCY : seen in the project join normal form



the most reliable and efficient database storage is stable storage .


THE MOST USEFUL DATABASE RECOVERY MECHANISM IS LOG BASED RECOVERY
log is a sequence of log records and keeps a recohrd of all database modification in the system. various types of log records available . no database modification can occur when log is in progress .
as the log is written onto stable storage and info. residing in stable storage is never lost theoretically speaking , most database system use log . actual recovery scheme depends on whether the immediate or deferred database of log based recovery is used


An alternative to log based recovery is SHADOW PAGING . Shadow paging is more faster than log based recovery but suffers from drawbacks like commit overhead , data fragmentation and garbage collection
